局部避障问题

在游戏里，一个角色通常已经有了全局路径，例如 A* 给出的导航路径。问题在于：多个角色沿着各自路径移动时，会在局部区域相互阻挡。

如果只让每个角色盯着自己的目标点走，就会出现拥挤、抖动、互相穿插等现象。局部避障算法要解决的是：在不重新规划整条路径的情况下，为角色选择一个当前帧更安全的速度。

RVO，全称 Reciprocal Velocity Obstacles，是一种多智能体局部避障方法。它建立在速度空间上，不直接修改位置，而是为每个 agent 选择新的速度。

从 VO 到 RVO

VO，即 Velocity Obstacle，用来描述一组会导致碰撞的速度。

假设 agent A 和 agent B 都可以近似成圆。如果 A 选择某个速度后，在未来时间范围内会和 B 相撞，那么这个速度就落入 A 关于 B 的速度障碍区域。

直观上，VO 的想法是：

$$ v_A - v_B \in VO_{A|B} $$

也就是说，真正决定碰撞趋势的是相对速度。

如果 A 只考虑避开 B 的 VO，它可能会做出非常大的转向；而 B 同时也可能做出类似动作，双方反而产生来回摆动。

RVO 的改进在于“互惠”：假设双方都会承担一部分避障责任。

RVO 的核心假设

RVO 将避障动作分摊给两个 agent。对于 A 来说，它不再假设 B 保持当前速度不变，而是假设 B 也会进行对称调整。

论文中给出的核心思想可以理解为：选择一个尽量接近期望速度、同时不会落入互惠速度障碍区域的新速度。

在游戏实现中，一般流程如下：

  1. 根据路径跟随或目标点计算期望速度。
  2. 查询附近 agent。
  3. 对每个邻居构造速度空间中的限制。
  4. 从候选速度中选择最接近期望速度且不碰撞的速度。
  5. 用新速度更新角色位置。

在游戏中的使用方式

RVO 适合处理大量移动单位的近距离避让，例如：

  • RTS 中士兵互相绕行。
  • 城市模拟中行人移动。
  • 多个 NPC 同时通过狭窄通道。
  • 联机游戏里非关键单位的视觉避障。

全局路径规划负责“往哪里走”，RVO 负责“这一小段怎么走得不撞”。两者通常组合使用。

实际项目中还需要考虑空间加速结构。每个 agent 不可能检查所有其他 agent，否则复杂度会接近 \(O(n^2)\)。常见做法是使用网格、四叉树、KD-Tree 等结构只查询附近邻居。

局限

RVO 的效果依赖局部假设。当人群特别密集、出口很窄、目标方向相互冲突时，角色可能仍然会出现停滞或局部拥堵。

此外，RVO 更关注运动学层面的速度选择，并不理解动画、角色朝向、攻击判定、队形保持等游戏语义。工程中经常需要额外加入优先级、半径膨胀、速度平滑和最大角速度限制。

References
