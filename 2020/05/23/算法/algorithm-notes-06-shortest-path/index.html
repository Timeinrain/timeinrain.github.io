从 BFS 到最短路

在无权图中，每条边的代价都可以看作 1，因此 BFS 能得到最短步数。

但当边有不同权重时，BFS 的层次就不再代表真实距离。此时需要专门的最短路算法。

Dijkstra

Dijkstra 适用于边权非负的单源最短路。

核心思想是：每次选择当前距离最小、尚未确定的点，将它作为中转点更新相邻点。

使用优先队列后，复杂度通常写作 \(O((n+m)\log n)\)。

Bellman-Ford

Bellman-Ford 可以处理负权边，并能检测负环。

它反复松弛所有边。若第 \(n\) 轮仍然可以更新，说明存在从源点可达的负环。

复杂度为 \(O(nm)\)，比 Dijkstra 慢，但适用范围更广。

Floyd

Floyd 用于多源最短路，即任意两点之间的最短距离。

它的思想是逐步允许更多中转点。外层 k 表示当前允许经过编号不超过 k 的点。

复杂度为 \(O(n^3)\)，适合点数较小的图。

如何选择

常见选择如下：

  • 无权图最短路：BFS。
  • 非负边权单源最短路：Dijkstra。
  • 有负权边：Bellman-Ford 或 SPFA 的谨慎变体。
  • 多源最短路且点数小：Floyd。

小结

最短路算法的关键是看边权条件。不要在负权图中直接使用 Dijkstra，因为它“确定当前最小点”的前提会被负边破坏。

从 BFS 到 Dijkstra，可以理解成从“每条边代价相同”扩展到“每条边代价不同，但都不为负”。
