前两篇解决“水面在哪里”。本篇解决“它为什么像水”。初版 shading 不追求一步到位，而是把法线、Fresnel、反射、折射、吸收、泡沫这些核心部分组织成可调、可降级、可调试的 UE 材质方案。

Shading输入

初版材质不要只依赖一张法线贴图。FFT 模拟已经给出了更有价值的数据：

输入 来源 用途
DisplacementTexture FFT post pass 顶点位移、局部高度、深度过渡
NormalTexture 斜率 IFFT 或高度差分 主法线和高光方向
FoamTexture Jacobian、曲率或浪尖阈值 白沫 mask
SceneDepth UE 场景深度 水深、岸边、折射遮挡
SceneColor 屏幕颜色 简化折射
Reflection SSR、Planar、Capture 或天空 水面反射

材质模型选择

UE5 项目优先考虑 Single Layer Water。Epic 文档中说明这个模型用于物理基础水面，能在单层深度近似下处理散射、吸收、反射、折射和阴影。它的渲染位置也比较特殊：材质可以用 Opaque 或 Masked blend mode，水的透明表现会在 base pass 和 deferred lighting 之后、常规 translucency 和 post process 之前执行。

如果是 UE4 或需要完全自定义，可以使用两种方案：

  • Translucent 材质：容易接 SceneColor 和 DepthFade，但排序、阴影、性能和反射控制更麻烦。
  • Opaque + 自定义水面合成：更像专用渲染 pass，控制力强，但工程复杂度高。

初版建议：

  • UE5：Single Layer Water + FFT 法线/位移/泡沫。
  • UE4：Translucent 或 Surface ForwardShading 材质先跑通，再按项目需要迁移到自定义 pass。

从 UE 源码看，Single Layer Water 至少要关注这些真实入口：

  • ShouldRenderSingleLayerWater 会检查 view 中是否有 Single Layer Water 材质和对应 mesh pass draw。
  • RenderSingleLayerWaterDepthPrepass 可在 base pass 前后生成水面深度。
  • AddCopySceneWithoutWaterPass 会准备 SceneColorWithoutSingleLayerWaterTexture 和 SceneDepthWithoutSingleLayerWaterTexture，供后续折射和合成使用。
  • RenderSingleLayerWaterInner 执行 EMeshPass::SingleLayerWaterPass。
  • RenderSingleLayerWaterReflections 接入 SSR、Lumen 或 reflection capture 一类反射路径。

这说明 FFT 海水的材质输入可以保持普通 UE 材质参数形式，但如果采用 Single Layer Water，最终 shading 会进入引擎专门的 water pass，而不是普通 translucent pass。

Fresnel和高光

Fresnel 是水感的骨架。初版可以用 Schlick 近似：

$$ F = F_0 + (1 - F_0)(1 - \max(0, N \cdot V))^5 $$

水的 \(F_0\) 通常很低，可以从 0.02 左右开始调。掠射角反射强，正视角更多看到水下颜色。

风越大，水面越碎，高光越宽；风越小，高光越尖。

反射

反射来源可以按成本分级：

  1. Planar Reflection：近景质量最高，成本也最高。
  2. SSR 或 Lumen reflection：适合屏幕内对象，但会有屏幕空间缺失。
  3. Reflection Capture：稳定便宜，适合中远景。
  4. Sky cubemap：远海和移动端兜底。

UE 的 Single Layer Water 源码里已经有一些值得借鉴的工程开关，例如 r.Water.SingleLayer.RefractionDownsampleFactor 控制折射相关纹理降采样，r.Water.SingleLayerWater.RefractionDistanceCulling、RefractionFresnelCulling 和 RefractionDepthCulling 控制折射剔除。自定义水面材质也可以复用这个思路：远距离、低 Fresnel 贡献或水深不足时，降低甚至跳过折射。

泡沫

泡沫可以先从三类 mask 合成：

  • 浪尖泡沫：来自 Jacobian 或曲率，表示波面压缩。
  • 岸边泡沫：来自水深，越浅越容易出现。
  • 交互泡沫：来自角色、船体或粒子写入的 render target。

Pass位置和调试

如果使用 Single Layer Water，要理解它不是普通 translucent。它会在 base pass 和 deferred lighting 之后参与水面合成，因此有些 SceneColor 或 depth 相关节点表现和普通半透明材质不同。调试时不要只看材质图，要结合 RenderDoc 或 UE 的 GPU Visualizer 看实际 pass。

初版调试建议：

  • ViewMode_NormalOnly：只看最终法线。
  • ViewMode_Fresnel：只看 Fresnel 权重。
  • ViewMode_Depth：只看水深。
  • ViewMode_ReflectionWeight：只看反射来源权重。
  • ViewMode_Foam：只看泡沫 mask。
