初版架构

基础版海水可以分成三层：

  • OceanSimulation：负责频谱更新、FFT、位移和法线贴图生成。
  • OceanSurface：负责提交海面网格，给材质提供海面贴图和参数。
  • OceanMaterial：负责最终反射、折射、泡沫和颜色。

在 UE 中，最推荐的边界是：模拟管线写在渲染线程，网格和材质参数由组件管理。

YivanLee 在 UE 中实现 FFT 海洋的文章采用的是更接近 UE4 的 RHI/RenderTarget 写法。UE5 项目可以把同样思路迁移到 RDG：将每一步计算表达成一个 pass，让图系统管理中间资源。

其他平台的实现也能提供很好的参照：NVIDIA OceanCS 把频谱更新、IFFT 和 displacement map 输出拆成明确的 GPU 阶段；Gikster 的实现文章把 displacement/slope 贴图、horizontal/vertical FFT 和最终 permutation 写成更接近伪代码的流程；WebGPU 实现则说明初始频谱可以初始化一次，而逐帧瓶颈主要落在时间频谱和 IFFT。

和 UE 源码对齐时，可以重点看两类结构：

  • RenderCore/Public/RenderGraphBuilder.h：FRDGBuilder::RegisterExternalTexture、AddPass、QueueTextureExtraction 是跨帧贴图和 RDG 临时资源之间的关键接口。
  • Renderer/Private/SingleLayerWaterRendering.cpp：引擎内置水体不是普通透明材质，而是通过 SingleLayerWaterDepthPrepass、SingleLayerWater、反射/折射复制和 tile classification 等步骤接入 deferred renderer。

因此自研 FFT 海水最好把“模拟贴图生成”和“水面材质渲染”分开：前者可以作为 RDG compute pass 生成稳定 render target，后者让 UE 的 mesh pass 或 Single Layer Water pass 负责真正 draw。

基础网格

基础网格用一个规则平面即可：

  • 分辨率：128 x 128 或 256 x 256。
  • 世界大小：与 FFT patch length 对齐，例如 512m、1024m。
  • 中心：每帧跟随相机的水平位置。
  • 顶点数据：只需要局部 xy 和 uv。

FoamRT 跨帧外部 RT 给材质采样

材质采样的贴图不要直接依赖 RDG 临时资源。做法是每帧在 RDG 末尾把结果写入注册过的外部 render target，然后动态材质实例持有这些 render target。

源码里 FRDGBuilder 提供了两条常见路径：

  • 已有跨帧 IPooledRenderTarget 时，用 RegisterExternalTexture 把它注册进当前 graph。
  • 当前帧由 RDG 创建、但下一帧还要继续使用时，用 QueueTextureExtraction 把结果提取到 TRefCountPtr<IPooledRenderTarget>。

海水模拟的 DisplacementRT、NormalRT、FoamRT 就适合按这种方式管理。FFT 中间 ping-pong 贴图仍然保持 RDG 临时资源，避免长期占用显存。

调试与最终效果

基础海水至少做四个调试开关：

  • ShowDisplacement：直接显示位移贴图。
  • ShowNormal：直接显示法线贴图。
  • ShowFoam：显示泡沫 mask。
  • FreezeTime：冻结频谱时间，排查空间采样问题。

最终效果检查顺序：

  1. 关闭 shading，只看 wireframe 和顶点位移。
  2. 关闭水平位移，只看高度是否稳定。
  3. 打开水平位移，检查浪尖方向。
  4. 打开法线，检查高光是否随波运动。
  5. 打开泡沫，检查浪尖是否出现白色区域。
  6. 打开反射和折射，检查是否有深度穿帮。

常见问题

海面出现棋盘格翻转

IFFT 后需要处理 \((-1)^{x+y}\) 的重排问题。可以在 post pass 中根据像素坐标奇偶乘以 -1。

波向和风向相反

检查频谱里的 \(\mathbf{k}\) 坐标、UE 世界坐标 X/Y、贴图 UV 方向是否一致。很多错误来自 Y 轴翻转。

材质采样到上一帧或空贴图

检查 RDG 外部资源导出时机。材质需要稳定的 render target 引用，中间临时资源不能跨帧直接使用。

海面边界能看到重复

这是基础平铺海面的天然问题。第四篇会把 Projected Grid、LOD、远近波形拆分和无限海洋作为优化项处理。
