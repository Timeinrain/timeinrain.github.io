排序解决的是什么

排序的直接目标是让元素按某种规则有序，但它更常见的作用是制造结构。

数组有序之后，很多问题会变简单：

  • 可以使用二分查找。
  • 可以用双指针从两端收缩。
  • 可以快速发现相邻元素之间的关系。
  • 可以把“无序枚举”变成“按顺序处理”。

因此排序往往不是终点，而是后续算法的准备阶段。

选择排序与插入排序

选择排序每一轮从未排序部分选出最小值，放到当前位置。

复杂度是 \(O(n^2)\)，优点是思路直接。

插入排序则像整理手牌：每次拿一个新元素，把它插入前面已经有序的部分。

插入排序最坏也是 \(O(n^2)\)，但当数组接近有序时表现很好。

归并排序

归并排序使用分治思想：

  1. 把数组分成左右两半。
  2. 分别排序。
  3. 合并两个有序数组。

每一层合并总成本为 \(O(n)\)，共有 \(\log n\) 层，因此复杂度是 \(O(n \log n)\)。

快速排序

快速排序选择一个基准值，把小于它的放左边，大于它的放右边，再递归处理左右两边。

平均复杂度为 \(O(n \log n)\)，但如果基准选择很差，最坏会退化到 \(O(n^2)\)。

它的优点是常数小、原地性好，因此工程中经常能看到快速排序思想的变体。

稳定性

如果两个元素关键字相等，排序后它们的相对顺序不变，就称排序是稳定的。

稳定性在多关键字排序时很有用。例如先按姓名排序，再按班级稳定排序，可以保留同班内姓名顺序。

常见结论：

  • 插入排序可以稳定。
  • 归并排序可以稳定。
  • 快速排序通常不稳定。
  • 堆排序通常不稳定。

小结

学习排序不能只背复杂度表。更重要的是理解三类思想：交换、插入、分治。

在实际使用中，优先调用语言标准库排序；在学习中，排序是理解递归、分治、稳定性和复杂度的好入口。
